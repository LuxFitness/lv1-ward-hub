requireAuth middleware:
```typescript
// backend/src/middleware/auth.ts
import { Request, Response, NextFunction } from 'express';

export function requireAuth(req: Request, res: Response, next: NextFunction) {
  if (req.session && (req.session as any).authenticated === true) {
    return next();
  }
  res.status(401).json({ error: 'Unauthorized' });
}
```

Rate limiter config (AUTH-03):
```typescript
// backend/src/middleware/rateLimiter.ts
import rateLimit from 'express-rate-limit';
import slowDown from 'express-slow-down';

export const loginRateLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 5,                    // 5 attempts per IP per window
  standardHeaders: true,
  legacyHeaders: false,
  message: { error: 'Too many login attempts, please try again later' },
});

export const loginSlowDown = slowDown({
  windowMs: 15 * 60 * 1000,
  delayAfter: 2,             // Start slowing after 2 attempts
  delayMs: (hits) => hits * 500,
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Express API | All unauthenticated requests cross here; requireAuth gates all non-auth routes |
| Express → Supabase | service_role key grants full DB access; must never reach browser |
| Login endpoint | Password hash comparison boundary; rate-limited to prevent brute force |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | POST /api/auth/login | mitigate | express-rate-limit (5/15min/IP) + loginSlowDown; bcrypt cost factor 12 (ASVS V2.1.1) |
| T-01-02 | Elevation of Privilege | express-session | mitigate | `req.session.regenerate()` after successful login (session fixation prevention); implemented in auth routes plan |
| T-01-03 | Tampering | CORS origin | mitigate | `cors({ origin: process.env.FRONTEND_URL })` — no wildcard; rejects cross-origin requests from unknown origins |
| T-01-04 | Info Disclosure | Error messages | mitigate | requireAuth returns generic 401; no stack traces in production (helmet default) |
| T-01-05 | Elevation of Privilege | service_role key | accept | Key stored in Render env vars only; never sent to browser; risk is acceptable for internal tool |
</threat_model>
